# Build failure search and categorization

This skill reads `*-bug.md` files (generated by the `launchpad-build-bug` skill)
and searches for common failure causes. It can produce a full taxonomy report
or filter by a specific failure category.

## Workflow

All paths below are relative to this skill directory.

### Mode 1 — Full taxonomy (summary)

Use when the user asks broadly about failure patterns, wants a top-level
overview, or says "categorize these."

```bash
python3 scripts/categorize_bugs.py <directory> --summary
```

`<directory>` is the path containing the `*-bug.md` files. Omit it to default
to the current working directory.

The script prints a human-readable markdown report grouped by category,
with counts and the title of each matching bug.

Surface the output **verbatim** to the user — don't summarize or compress it.
Add a final line noting the file the taxonomy reference lives in
("Full category definitions in `skills/build-failure-search/references/failure-categories.md`.")

### Mode 2 — Query by category (JSON + filtering)

Use when the user asks about a specific failure pattern, e.g.
"show me all cmake bugs" or "how many gcc-15 failures are there."

```bash
python3 scripts/categorize_bugs.py <directory>
```

This prints the full JSON object: a `categories` dictionary (label → human-readable description) and a `bugs` array with each bug's metadata, failure text, likely cause, and assigned categories. Pipe through `jq` to filter.

Example — list all bugs in the `c23-keyword` category:

```bash
python3 scripts/categorize_bugs.py <directory> | python3 -c "
import json, sys
d = json.load(sys.stdin)
for b in d['bugs']:
    if 'c23-keyword' in b['categories']:
        print(b['title'])
"
```

Example — show the Failure section for all `gcc-warnings-as-errors` bugs:

```bash
python3 scripts/categorize_bugs.py <directory> | python3 -c "
import json, sys
d = json.load(sys.stdin)
for b in d['bugs']:
    if 'gcc-warnings-as-errors' in b['categories']:
        print('###', b['title'])
        print(b['failure'], '\n')
"
```

After running a query, show the user the matching results. If the user asked
for a specific field (titles, counts, failure excerpts), surface only that.
Otherwise show title + failure for each match.

### Mode 3 — Search for a custom keyword/pattern

Use when the user asks about a failure cause not covered by the built-in
categories, e.g. "find bugs related to openssl" or "show me all pthread
issues."

Use the `Grep` tool to search across the .md files:

- For listing matching files: `pattern="<pattern>" path="<directory>" include="*-bug.md"`
- For reading matches with context, `Grep` returns file:line content.

Present matching file names with their titles.

## Reference

See `references/failure-categories.md` for the full taxonomy of failure
categories, pattern descriptions, and the frequency distribution from
the resolute test-rebuild dataset.

## When NOT to use this skill

- If the user shares a **URL** (Launchpad build page or buildlog link) —
  use `launchpad-build-bug` instead. This skill operates on already-generated
  bug reports, not on raw buildlogs.
- If the user is asking about a **single** bug report — just `Read` the file.
- If the user wants to **edit** or **fix** a bug report — that's out of scope.
